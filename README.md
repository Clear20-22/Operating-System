# Operating System Lab

This repository contains materials for my Operating System lab work. It is mainly organized around an OS/161-based setup, including the source tree, simulator, and related toolchain components used for lab practice and experimentation.

## Repository Purpose

This repo is intended for:

- operating system lab assignments
- OS/161 practice and experiments
- kernel and system-level development work
- keeping lab files and source code in one place

## Project Structure

The main contents are inside the `Demo OS/` directory:

- `Demo OS/os161-1.99/` - OS/161 source code
- `Demo OS/sys161-1.99.06/` - System/161 simulator
- `Demo OS/gcc-4.1.2+os161-2.0/` - GCC toolchain source for OS/161
- `Demo OS/gdb-6.6+os161-2.0/` - GDB source for debugging
- `Demo OS/binutils-2.17+os161-2.0.1/` - binutils source
- `Demo OS/bmake/` - build tool components
- `Demo OS/mk/` - make/build configuration files

## Notes

- This repository is for academic and lab use.
- Build outputs, temporary files, and common generated artifacts are ignored through `.gitignore`.
- Additional lab reports, notes, or assignment-specific files can be added later as needed.

## Getting Started

1. Clone the repository.
2. Open the `Demo OS/` directory and identify the component you want to work on.
3. Use the OS/161 and System/161 tools according to your lab instructions.
4. Keep your code, experiments, and lab updates version controlled in this repo.
